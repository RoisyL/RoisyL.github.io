目录结构

初学者常操作文件

关键目录/文件 说明
config 配置目录，包括端口号等。我们初学可以使用默认的。
node_modules npm 加载的项目依赖模块
src 这里是我们要开发的目录，基本上要做的事情都在这个目录里。
•assets: 放置一些图片，如logo等。
• components: 目录里面放了一个组件文件，可以不用。
• App.vue: 项目入口文件，我们也可以直接将组件写这里，而不使用 components 目录。
• main.js: 项目的核心文件。
• index.css: 样式文件。
static 静态资源目录，如图片、字体等。
test 初始测试目录，可删除
index.html 首页入口文件，可以添加一些 meta 信息或统计代码
package.json 项目配置文件。
dist 使用 npm run build 命令打包后会生成该目录。
  1. main.js - 这是 Vue 应用的入口文件。在这个文件中，你通常会进行以下几个操作：
    • 引入 Vue 库、插件（如 Vuex, Vue Router 等）
    • 创建 Vue 实例。
    • 挂载根组件（App.vue）。
    • 配置全局混入（mixins）或指令（directives）。
  2. App.vue - 这是根组件，它是所有其他组件的父级。在这个文件中，你可以定义全局的布局和样式。
  3. components 目录 - 这个目录用于存放项目中使用的所有 Vue 组件。每个组件通常都有自己的 .vue 文件。
  4. views 目录 - 这个目录通常包含一些页面级别的组件，它们通常与 Vue Router 路由相匹配。例如，你可能会有一个 Home.vue 组件对应于主页的路由。
  5. router.js 或 router/index.js - 如果你的应用是一个单页面应用（SPA），并且使用了 Vue Router，那么这个文件用于定义应用的路由。在这里，你可以声明路由路径、视图组件以及路由的嵌套路由。
  6. store.js 或 store/index.js - 如果你的应用需要状态管理，并且使用了 Vuex，那么这个文件或目录用于定义应用的状态。在这里，你可以定义状态、变更状态的 mutation、以及可以触发 mutation 的 action。
  7. assets 目录 - 这个目录位于 src 目录下，用于存放应用中使用的静态资源，如图片、样式表、字体等。这些资源可以在你的组件中通过相对路径引用。

main.js

  • createApp 函数：创建一个新的 Vue 应用实例。这个实例包括了应用的响应式数据、指令、组件、生命周期钩子等核心功能，通过这个实例可以协调和管理整个 Vue 应用的生命周期和功能
  • 挂载：是 Vue 应用生命周期的一个重要阶段，它标志着应用实例开始与 DOM 交互，应用的渲染和用户交互功能正式开始工作
  • 注册：指的是将组件、指令、混入（mixins）或其他资源添加到 Vue 应用实例或 Vue 配置全局中，使得它们可以在应用的任何地方使用或生效
    • 允许开发者根据需要添加功能和资源，而不是强制所有功能都默认可用。这种方式提高了 Vue 的可扩展性和模块化程度

index.html

体现vue特点的关键代码：

这段代码的功能是初始化 Vue 应用并将其挂载到页面上：

  1. 页面加载时，浏览器首先解析 index.html 文件，并在 <body> 标签中找到 <div id="app"></div> 元素。
  2. 浏览器接着加载并执行 <script> 标签指定的 main.js 文件。
  3. main.js 文件中的代码创建了一个 Vue 应用实例，并调用 .mount('#app') 方法将该实例挂载到 <div id="app"></div> 元素上。
  4. Vue 应用实例开始运行，根据根组件的模板和数据渲染页面内容，替换 <div id="app"></div> 元素的初始内容。

通过这种方式，Vue 应用可以接管页面的 <div id="app"></div> 部分，并动态地根据数据变化更新页面内容，提供丰富的用户交互和体验。
