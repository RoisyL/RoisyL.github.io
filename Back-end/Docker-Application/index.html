  • 启动容器
参数 解释
-d 后台运行
--name mysql-icwp-alone 设置容器名字
-p 3306:3306 端口映射
-e TZ=Asia/Shanghai 设置mysql时区
-e MYSQL_ROOT_PASSWORD=123456 设置mysql密码
-v ./data:/var/lib/mysql 挂载mysql数据
-v ./conf:/etc/mysql/conf.d 挂载配置文件
-v ./init:/docker-entrypoint-initdb.d 挂载初始化.sql文件
-v ./log:/var/log/mysql 挂载log文件
--network icwp-alone 将容器加入网络（方便后续用容器名字代替ip来访问mysql）
mysql:8.0 容器所基于的镜像名字

Django容器化

基础文件结构：

  • 数据迁移指令

    • 作用：确保数据库结构与代码库中（models.py）的模型定义保持同步
    • 在部署新版本的 Django 应用时，通常还会使用 python manage.py collectstatic 命令，以确保所有静态文件被正确收集和设置

  • 如果需要用到docker-compose，那根目录下还需要编写docker-compose.yml文件

Dockerfile

  • FROM python:3.10 指令从仓库拉取一个包含 python 3.10 的 Linux 操作系统环境
  • RUN 和 WORKDIR 指令都是针对容器的，功能是在容器里创建目录、并将其设置为工作目录。注意宿主机是没有这个目录的。
  • ADD 指令出现了两次，都起复制文件的作用

requirements.txt

settings.py

  • 将数据库设置为 MySQL

    • 后端连接不上数据库的可能原因：

      不能直接用localhost访问数据库，而要用后端配置中 depends_on 后的容器名称（即数据库的容器名称）加上端口去访问

      因为在容器内部，localhost 通常指向容器自身，而不是宿主机或其他容器

  • 设置调试模式

    在开发环境中，一般将 DEBUG 设置为 True 以获取详细的错误信息

    而在生产环境中则设置为 False 以提高安全性和性能

  • 明确允许哪些域名或IP地址可以访问网站

  • 定义静态文件的 URL 前缀和根目录

    • STATIC_ROOT 是静态文件的本地根目录，用于存放通过 collectstatic 命令收集的静态文件
